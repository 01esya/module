# История изменений (Changelog)

Все значимые изменения в проекте **CargoFlow** документируются в этом файле. Проект следует принципам семантического версионирования (Semantic Versioning).

---

## [1.1.2] — 2026-06-19

- Создано подробное руководство разработчика [DEVELOPER_GUIDE.md](file:///DEVELOPER_GUIDE.md) по развертыванию, конфигурации переменных окружения, тестированию и Docker.
- Добавлена интерактивная анимированная кнопка-уголок GitHub (GitHub Corner Ribbon) на страницу презентации для удобного перехода в репозиторий.

### Исправлено
- Отредактировано форматирование разметки Markdown в описании схемы СУБД и распределенной архитектуры данных (MDM).
- Ссылки на файлы документации в [README.md](file:///README.md) изменены с абсолютных локальных на относительные для корректного отображения на GitHub.

## [1.1.0] — 2026-06-16
### Добавлено
- Оркестрация контейнеров через `docker-compose.yml` в корне проекта (совместный запуск бэкенда, раздача статики фронтенда и сохранение SQLite в volume).
- Интеграция конфигурационных параметров в класс `Settings` на базе `Pydantic Settings` (включая безопасность Cookie сессии).
- Автоматический отказоустойчивый режим (API Fallback) для справочников Supabase: при сбое подключения к удаленному серверу бэкенд автоматически переходит на локальную копию данных SQLite.

### Исправлено
- Ошибка прав доступа (Permission Denied) при записи базы SQLite внутри Docker-контейнера. Добавлена корректная инициализация директории `/app/data` и флаги смены владельца `--chown=cargoflow:cargoflow` в Dockerfile.
- Проблема жестко захардкоженного адреса `http://localhost:8000` в панели аналитики фронтенда. URL заменен на относительные запросы для универсальной работы во всех контурах (Docker, Local).
- Исправлено наложение переносов строк (LF/CRLF) в конфигурационных файлах.

## [1.0.0] — 2026-06-10
### Добавлено
- Базовая версия клиент-серверного веб-приложения CargoFlow.
- Модуль создания и печати электронных путевых листов (ЭПЛ) по стандартам Минтранса РФ.
- Модуль интерактивной карты (GIS) с использованием библиотеки **OpenLayers** и карт **OpenStreetMap** для отображения ТС.
- Симулятор телеметрии (скорость, уровень топлива, GPS координаты) с автообновлением каждые 12 секунд.
- ИИ-ассистент на базе больших языковых моделей (LLM через OpenRouter) для аудита безопасности рейсов.
- Локальная база данных SQLite для транзакций и сотрудников, внешняя база данных Supabase для общего реестра ТС.
