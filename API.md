# Документация REST API — CargoFlow Backend

Серверный модуль CargoFlow предоставляет программный интерфейс (REST API) для управления сотрудниками, путевыми листами, получения данных телеметрии транспортных средств и проведения ИИ-аудита безопасности рейсов.

Все эндпоинты, кроме проверки здоровья сервера, требуют аутентификации.

---

## 🔐 Аутентификация и безопасность

Аутентификация построена на базе JWT-токенов.
- При авторизации токен записывается в безопасную **HttpOnly Cookie** с именем `sb_access_token`.
- Также поддерживается передача токена в заголовке `Authorization: Bearer <token>`.
- Настроена защита от спама (Rate Limiting) — максимум **60 запросов в минуту** с одного IP.

## 📡 Список эндпоинтов

### 1. Сервисные службы (Health)

#### `GET /health`
- **Описание:** Проверка работоспособности сервиса.
- **Требует авторизации:** Нет.
- **Ответ:**
  ```json
  {
    "status": "healthy",
    "version": "1.0.0"
  }
  ```

---

### 2. Аутентификация (Auth)

#### `POST /api/auth/login`
- **Описание:** Авторизация пользователя по Email и паролю. Устанавливает cookie `sb_access_token`.
- **Запрос (JSON):**
  ```json
  {
    "email": "user@example.com",
    "password": "password123"
  }
  ```
- **Ответ (200 OK):**
  ```json
  {
    "access_token": "jwt_token_here",
    "token_type": "bearer",
    "user": {
      "id": 1,
      "email": "user@example.com",
      "role": "dispatcher",
      "organization_id": 1
    }
  }
  ```

#### `POST /api/auth/logout`
- **Описание:** Выход из системы. Удаляет cookie `sb_access_token`.
- **Ответ:**
  ```json
  {
    "detail": "Logged out"
  }
  ```

#### `GET /api/auth/me`
- **Описание:** Получение профиля текущего авторизованного пользователя.
- **Ответ:**
  ```json
  {
    "id": 1,
    "email": "user@example.com",
    "role": "dispatcher",
    "organization_id": 1
  }
  ```

---

### 3. Электронные путевые листы (Waybills)

#### `GET /api/waybills`
- **Описание:** Список путевых листов текущей организации.
- **Параметры запроса (Query):**
  - `status` (опционально): Фильтр по статусу (`Ожидают`, `В пути`, `Доставлен`, `Отменён`).
- **Ответ:** Список путевых листов (массив JSON-объектов).

#### `POST /api/waybills`
- **Описание:** Создание нового электронного путевого листа (Форма № 4-П).
- **Запрос (JSON):**
  ```json
  {
    "cargo_type": "Строительные материалы",
    "weight": 1250.5,
    "customer": "ООО МеталлСнаб",
    "carrier": "ИП Иванов",
    "from_city": "Москва",
    "to_city": "Санкт-Петербург",
    "date_from": "2026-06-17",
    "date_to": "2026-06-19",
    "vehicle_id": 2,
    "driver_id": 5,
    "route_coords": [
      [55.7558, 37.6173],
      [59.9343, 30.3351]
    ],
    "organization_id": 1
  }
  ```
- **Ответ (201 Created):** Объект созданного путевого листа с присвоенным `id` и `status="Ожидают"`.

#### `GET /api/waybills/{waybill_id}`
- **Описание:** Получение детальной информации по конкретному путевому листу.
- **Ответ:** Данные путевого листа.

#### `PATCH /api/waybills/{waybill_id}`
- **Описание:** Частичное обновление полей путевого листа.
- **Запрос:** Содержит только обновляемые поля (в формате JSON).

#### `PATCH /api/waybills/{waybill_id}/status`
- **Описание:** Изменение статуса рейса.
- **Запрос (JSON):**
  ```json
  {
    "status": "В пути"
  }
  ```

#### `DELETE /api/waybills/{waybill_id}`
- **Описание:** Удаление путевого листа.
- **Ответ:** 204 No Content.

### 4. Управление персоналом (Employees)

#### `GET /api/employees`
- **Описание:** Список водителей и диспетчеров организации.
- **Ответ:** Список сотрудников.

#### `POST /api/employees`
- **Описание:** Регистрация нового сотрудника в картотеку.
- **Запрос (JSON):**
  ```json
  {
    "first_name": "Иван",
    "last_name": "Петров",
    "middle_name": "Сергеевич",
    "role": "driver",
    "license_number": "99 12 345678",
    "organization_id": 1
  }
  ```

#### `PATCH /api/employees/{employee_id}`
- **Описание:** Редактирование карточки сотрудника.

#### `DELETE /api/employees/{employee_id}`
- **Описание:** Удаление сотрудника из системы.

---

### 5. Транспортные средства (Vehicles)

#### `GET /api/vehicles`
- **Описание:** Список транспортных средств компании.
- **Специфика:** Данные запрашиваются из централизованного реестра Supabase. При сбое сети бэкенд автоматически возвращает локально сохраненный список ТС (SQLite реплика) для непрерывности работы.
- **Ответ:** Массив транспортных средств (марка, госномер, грузоподъемность и т.д.).

---

### 6. Телеметрия и мониторинг (Monitoring)

#### `GET /api/monitoring/vehicles`
- **Описание:** Текущие GPS/ГЛОНАСС координаты и данные датчиков всех активных ТС компании.
- **Ответ:** Словарь, где ключ — ID машины, а значение — объект телеметрии (скорость, уровень топлива, спутники, широта, долгота).

#### `GET /api/monitoring/vehicles/{vehicle_id}`
- **Описание:** Координаты и датчики конкретного автомобиля.

---

### 7. ИИ-ассистент и аудит (AI)

#### `POST /api/ai/analyze`
- **Описание:** Запуск ИИ-аудита рейса на основе путевого листа и собранной телеметрии. Использует OpenRouter API для отправки контекста в LLM.
- **Запрос (JSON):**
  ```json
  {
    "waybill_id": 12,
    "question": "Проведи аудит рейса: оцени скорость, соблюдение маршрута и состояние автомобиля."
  }
  ```
- **Ответ (200 OK):**
  ```json
  {
    "report": "Анализ рейса № 12 показывает следующее...\nРиски: превышений скорости не обнаружено...\nРекомендации: ..."
  }
  ```
