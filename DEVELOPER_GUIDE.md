# 💻 Руководство разработчика — CargoFlow Terminal

Это руководство содержит подробные инструкции по локальному развертыванию, настройке переменных окружения, тестированию, сборке и развертыванию информационной системы CargoFlow.

---

## 📋 Системные требования

Перед началом установки убедитесь, что на вашей рабочей станции установлены:
* **Node.js** версии `18.x` или выше (рекомендуется LTS).
* **Python** версии `3.12.x` или выше.
* **Docker** и **Docker Compose** (опционально, для контейнеризации).
* Клиент контроля версий **Git**.

---

## 🛠️ Локальное развертывание среды разработки

Разработка ведется в режиме раздельного запуска фронтенд-сервера и бэкенд-сервера с проксированием запросов.

### Шаг 1. Настройка Бэкенда (FastAPI)

1. В терминале перейдите в каталог бэкенда:
   ```bash
   cd backend
   ```

2. Выберите блок команд под вашу операционную систему, скопируйте его целиком и выполните в терминале для создания виртуального окружения и установки библиотек:

   * **Для Windows (PowerShell):**
     ```powershell
     python -m venv .venv
     .\.venv\Scripts\Activate.ps1
     pip install -r requirements.txt
     ```

   * **Для Linux / macOS:**
     ```bash
     python -m venv .venv
     source .venv/bin/activate
     pip install -r requirements.txt
     ```

## 🔑 Конфигурация переменных окружения (`.env`)

Создайте в папке `backend/` файл `.env` (на базе примера `.env.example`). 

> [!WARNING]
> **ПРАВИЛО БЕЗОПАСНОСТИ:** Ни в коем случае не коммитьте файл `.env` с заполненными реальными ключами доступа и не отправляйте его на GitHub! Этот файл занесен в `.gitignore` и должен существовать только локально на сервере/компьютере разработчика. В репозиторий загружается только шаблон `.env.example`.

Заполните переменные шаблона вашими локальными значениями:

```env
# Параметры подключения к облачной СУБД Supabase
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_EMAIL=your_service_email
SUPABASE_SERVICE_PASSWORD=your_service_password

# Токен доступа к LLM-моделям для ИИ-ассистента
OPENROUTER_API_KEY=your_openrouter_api_key

# Настройки безопасности бэкенда
JWT_SECRET=your_jwt_secret_key_minimum_32_characters
APP_ENV=development
BACKEND_PORT=8000
FRONTEND_ORIGIN=http://localhost:5173
```

## ⚡ Шаг 2. Инициализация и запуск Бэкенда

При первом запуске сервер FastAPI автоматически проверит наличие файла базы данных SQLite (`backend/cargoflow.db`). Если файла нет, система:

1. Создаст схему базы данных.
2. Применит все миграции.
3. Проведет **seed-заполнение** базы данных тестовыми записями (создаст пользователя-диспетчера по умолчанию, тестовых водителей и путевые листы).

Запустите бэкенд-сервер командой:
```powershell
.\run_server.ps1
```

*Интерактивная документация Swagger UI станет доступна по адресу:* `http://localhost:8000/docs`

## 🖥️ Шаг 3. Настройка и запуск Фронтенда (React + Vite)

1. Вернитесь в корень репозитория:
   ```bash
   cd ..
   ```

2. Установите npm-пакеты и запустите сервер разработки фронтенда:
   ```bash
   npm install
   npm run dev
   ```

*Веб-интерфейс терминала откроется по адресу:* `http://localhost:5173`

## 🧪 Запуск автоматических тестов

Проект укомплектован автоматическими тестами серверной логики (интеграционные тесты API, проверка авторизации, ИИ-модуля и CRUD операций):

```powershell

```

Для проверки статической типизации и линтинга фронтенда из корня проекта:
```bash
npm run lint
```

## 🐳 Контейнеризация и оркестрация (Docker Compose)

Для быстрого запуска всей инфраструктуры в продакшен-режиме (когда бэкенд раздает и API, и скомпилированную статику React):

1. Выполните сборку фронтенда в папку `dist/`:
   ```bash
   npm run build
   ```

2. Запустите сборку и старт контейнеров:
   ```bash
   docker-compose up --build
   ```

*Приложение будет доступно на порту:* `http://localhost:8000`

## 🩺 Решение частых проблем (Troubleshooting)

### 1. Ошибка блокировки базы данных SQLite
* **Ошибка:**
  ```text
  sqlite3.OperationalError: database is locked
  ```
* **Причина:** Блокировка транзакций SQLite при одновременной записи из разных потоков (или зависший процесс бэкенда).
* **Решение:** Перезапустите бэкенд-сервер с помощью скрипта `.\run_server.ps1`, который автоматически завершает зависшие фоновые процессы Python.

### 2. Ошибка соединения с Supabase
* **Ошибка:** Превышение таймаута при запросе к внешнему API.
* **Причина:** Отсутствие интернет-соединения или неверные ключи в файле `.env`.
* **Решение:** Проверьте правильность `SUPABASE_URL` и `SUPABASE_ANON_KEY`. В случае отсутствия сети бэкенд автоматически переходит в режим *offline-fallback* и подгружает данные из локального SQLite-кэша.
