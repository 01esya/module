Информационная система формирования электронных путевых листов с модулем спутникового мониторинга ТС и ИИ-аудитом безопасности рейсов

Ключевой функционал системы

Электронные путевые листы (ЭПЛ)

Выписка и ведение путевых листов по форме Минтранса РФ № 4-П. Поддержка валидации, смены статусов рейса и вывод печатных версий документов в формат PDF.

Спутниковый GPS/ГЛОНАСС мониторинг

Интеграция с навигационным оборудованием. Отображение транспортных средств на интерактивной ГИС-карте (OpenLayers) с автоматическим расчетом трасс OSRM.

ИИ-аудит рейсов и LLM-аналитика

Проверка безопасности выполнения рейсов на базе искусственного интеллекта (через API OpenRouter). Оценка рисков перерасхода топлива и усталости водителя.

Отказоустойчивая архитектура (Fallback)

Использование централизованных мастер-данных в облаке Supabase с автоматическим прозрачным переключением на локальный SQLite кэш при потере интернет-соединения.

Роль: Диспетчер (поля ВУ скрыты)

7. Печатная форма путевого листа (Типовая форма № 4-П)
